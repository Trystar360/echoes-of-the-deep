# Showcase World — design & build plan

A guided, walkable **tutorial world** that demonstrates every system in *Octaves of the
One*, in the spirit of modpack showcase worlds (Create's "Above & Beyond" tutorial wing,
Mekanism demo halls, the GregTech "guide" maps). It mirrors the in-game
[Great Work](wiki/The-Great-Work.md) advancement tree so the world and the advancements
teach the same path.

> **Status: planned, not built.** This document is the spec. Building it is a follow-up —
> the world is constructed programmatically (see *Build pipeline*) and shipped as a
> packaged save. It also becomes the set we use to capture real in-game screenshots for
> the Modrinth gallery.

## Goals

1. **Teach the loop, in order** — refine → generate → carry/bank → spend → radiation →
   wireless → octave climb → transmutation → garden. One linear path, no dead ends.
2. **Show, don't tell** — every station is a *working* mini-build the player can watch run,
   not just a block on a pedestal. Signage explains the "why" in the mod's voice.
3. **Be reproducible** — the whole world is generated by a datapack of functions, so it can
   be rebuilt, version-bumped, and extended by modpacks. The binary save is a convenience
   artifact derived from it.

## Delivery format

- **`echoes_showcase` datapack** (in-repo, under `showcase/datapack/`) — `mcfunction`
  files that build the hall via `/fill`, `/setblock`, `/summon`, and `/data`.
- **Packaged save** — `Octaves-of-the-One-Showcase.zip` (a superflat creative world with
  the hall pre-built), attached to the GitHub Release and linked from the wiki. This is the
  one-click experience for players and the screenshot set for the gallery.
- The datapack stays the source of truth; the save is regenerated from it by the build
  pipeline below.

## World settings

- **Superflat** (a single grass layer over dirt/bedrock) so the architecture reads clean.
- Gamerules baked into the save: `doDaylightCycle false` (fixed pleasant daytime for
  screenshots; one wing temporarily sets night/storm for the Storm Caller), `doWeatherCycle
  false`, `doMobSpawning false`, `keepInventory true`, `mobGriefing false`,
  `doTileDrops true`. Difficulty peaceful.
- Spawn at the **Atrium** entrance, facing down the main axis. World spawn + a respawn
  anchor/bed so the player always returns to the start.
- Creative-mode world; a chest at spawn also holds one of every mod item for free
  experimentation, with a sign noting survival recipes live on the wiki / in advancements.

## Layout

A straight central concourse running +X, made of **Echocite Bricks** with **Lume Lantern**
lighting and a **Lumewood** trim, so the building set is itself on display. Wings open off
the concourse at fixed intervals. Each wing is a `16×16` bay with a title arch.

```
            N
            │
 ┌──────────┴───────────────────────────────────────────────────────────┐
 │ Atrium → W1 Refine&Gen → W2 Wired Grid → W3 Radiation → W4 Wireless →  │  +X →
 │          → W5 Octave Climb → W6 Transmutation → W7 The Grove → Gear     │
 └────────────────────────────────────────────────────────────────────────┘
```

Origin is `(0, -60, 0)` (build at a fixed, documented anchor so functions use absolute
coordinates and are idempotent — re-running rebuilds cleanly). Each wing starts every 24
blocks along +X; the table below is the authoring grid.

| Wing | Start X | Teaches |
| --- | --- | --- |
| Atrium | 0 | Welcome, how to use the world, open advancements (`L`) |
| W1 Refine & Generate | 24 | Echocite → Echo Ingot; Resonant Coil, Stillness Core, Octave Coil, Storm Caller |
| W2 The Wired Grid | 48 | Conduits, Resonance Cells, Compressor, Transmuter, Balancer |
| W3 Radiation | 72 | Growth & Warmth Radiator, Polarity Field |
| W4 Wireless Transport | 96 | Channels, Wave Relay + family, Tuner & Atlas |
| W5 The Octave Climb | 120 | Silentite, Octave Seed, Radiant chain, Greater Cell, Octave tiers |
| W6 Transmutation | 144 | Table & Tablet, the Mote ladder, Octave Stars |
| W7 The Octave Grove | 168 | Lumewood set, Lumebloom, Lume Lantern, Verdant Loam, masonry |
| Gear Alcove | 192 | Resonant tools + a Thruster flight course |

## Teaching method (per station)

Every station combines three elements:

1. **A working demo.** Examples:
   - W1: a Resonant Coil ringed by note blocks on a redstone clock — comparator → lamp
     shows its buffer filling from ambient sound, live.
   - W2: Coil → Wave Conduit → Resonance Cell → Compressor with a hopper auto-feeding
     cobblestone, so the player watches ore-doubling happen.
   - W4: two Wave Relays on the same channel (one Send, one Receive) beaming items across a
     glass gap, with a Wave Chest buffering.
   - W3: a Growth Radiator over tilled, planted farmland visibly accelerating crops.
2. **A title + lore sign** (front-text NBT) naming the block in display style and giving the
   one-line cosmology framing (reuse the `tooltip.echoes.desc.*` lang lines).
3. **A lectern + written book** per wing with the deeper "how it works / how to build it in
   survival" explanation, cross-referencing the relevant advancement and wiki page.

Item frames (or glow item frames) label each block and show the **item form + recipe
result**, so the station doubles as a visual recipe reference. The Mote ladder (W6) is a row
of five framed Motes with their Light Values on signs.

## Datapack structure

```
showcase/datapack/
  pack.mcmeta
  data/echoes_showcase/function/
    build/all.mcfunction        # calls every wing in order, then sets spawn + gamerules
    build/atrium.mcfunction
    build/w1_refine.mcfunction
    build/w2_grid.mcfunction
    build/w3_radiation.mcfunction
    build/w4_wireless.mcfunction
    build/w5_octave.mcfunction
    build/w6_transmute.mcfunction
    build/w7_grove.mcfunction
    build/gear.mcfunction
    lib/concourse.mcfunction    # shared floor/wall/lighting helpers
    reset.mcfunction            # /fill air over the build volume to rebuild from scratch
```

`build/all` is idempotent: it clears the build volume then constructs everything, so it can
be re-run after edits. Signs/books store their text inline in the functions (kept readable
with one station per block of commands).

## Build pipeline (how the save is produced)

Runs entirely on the **headless dedicated server** — no rendering required:

1. Generate a superflat world with the `echoes_showcase` datapack present
   (`run/world/datapacks/`), `level-type=flat`, `online-mode=false`, in `eula`-accepted
   `run/`.
2. Start the server, run `/function echoes_showcase:build/all` from the console, confirm the
   log shows no command errors and the expected block counts (a `/fill ... replace` count
   check per wing).
3. `/save-all flush`, stop the server, then **zip `run/world`** into
   `Octaves-of-the-One-Showcase.zip`.
4. Attach the zip to the GitHub Release and link it from the wiki's Getting Started page.

A small wrapper script (`showcase/build_showcase.sh`) will automate steps 1–3 so the save
is reproducible from the datapack.

## Verification

- Server console: `/function` returns success and the per-wing block-count assertions match.
- Load the produced save on a client (manual, by the maintainer) to sanity-check layout and
  capture gallery screenshots.
- The datapack is plain data and validates with the same JSON/idempotency checks used for
  advancements.

## Screenshots (follow-up)

This world is the capture set for the Modrinth gallery: once built, load it on a real
client and shoot each wing (Atrium hero shot, the live wired grid, the wireless beam, the
Transmutation Table, the Grove). Until then the gallery uses generated **promo graphics**
(see `promo/` and `gen_promo.py`).

## Out of scope (v1)

- No custom dimension or terrain — superflat keeps the focus on the builds.
- No forced progression/quests (that's what the advancement tree is for); the world is a
  free-roam guided hall.
- Hand-tuned interior decoration beyond the standard station kit — can be layered on later.
